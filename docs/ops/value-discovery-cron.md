# Value Discovery Cron

Kör återkommande, till exempel veckovis, och uppdaterar docs/product/value-discovery-report.md.
Granskar repot ur kundvärde, inte teknik. Var direkt och skeptisk, prisa inte som default,
föredra att ta bort friktion framför att lägga till funktion, föredra bevis framför antaganden.

## Kärnfråga
Hjälper detta repo målanvändaren att nå ett meningsfullt resultat snabbare, lättare, billigare eller bättre?

## Frågor cronen besvarar
1. Vad är det tydligaste användarvärdet i repot just nu?
2. Vilket användarproblem löser repot?
3. Är problemet smärtsamt, frekvent eller värdefullt nog?
4. Vilka features stödjer det värdet direkt?
5. Vilka features ser ut som komplexitet, fåfänga eller distraktion?
6. Var är användarresan fortfarande förvirrande?
7. Vad är den snabbaste nästa förbättringen som ökar kundvärde?
8. Vad bör tas bort, förenklas eller skjutas upp?
9. Vilka bevis har vi från användning, feedback, analys eller tester?
10. Vilka bevis saknas?
11. Hur stödjer repot den långsiktiga artist-OS-visionen?
12. Vad bör nästa agent fokusera på?

## Om något saknas
Saknar repot tydlig målanvändare, tydligt flöde eller mätbart användarresultat: markera som kritiskt.
